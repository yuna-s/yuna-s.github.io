What I learned building my first GitHub Pages site

Turns out putting something on the internet is both easier and scarier than I expected. Here's how it went.

I've had a GitHub account for a while, but I'd never actually published anything with it. The repos were all private, half-finished projects I was too embarrassed to share. So when I decided to make a personal site, GitHub Pages felt like the right push.

It's just files

The thing that surprised me most: it really is just files. One index.html, pushed to a repo named username.github.io, and suddenly you have a website. No backend, no server costs, no setup beyond what you already know.

That simplicity is both freeing and a little overwhelming. There's no framework telling you what to do. You make the choices.

The scary part

The scary part isn't the tech. The scary part is that once it's live, anyone can see it. That's the whole point, but it's also terrifying in a way that a private repo never is.

Publishing is a different kind of finishing. It says: this is real, this counts, I made this.

I stared at the "push" button for longer than I'd like to admit.

What I'd tell myself before starting

  • Keep it simple. One HTML file is enough to start.
  • Separate your CSS early — it's much easier to manage.
  • Don't wait until it's perfect. It never will be.
  • The best time to publish was yesterday. The second best time is now.

The site you're reading right now is the result of that push. It's not finished. It might never be finished. But it's here, and that feels good.
