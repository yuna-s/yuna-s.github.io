## サイト構成

```
yuna-s.github.io/
├── index.html              ← ホームページ（About + Blog一覧）
├── css/
│   └── style.css           ← 全ページ共通スタイル
├── posts/
│   ├── _template.html      ← 新記事テンプレート
│   └── YYYY-MM-DD-*.html   ← 各記事
├── assets/
│   └── images/             ← 画像置き場
└── .claude/
    └── launch.json         ← プレビューサーバー設定（python http.server）
```

## スタイルの方針

- **テーマ**：warm（温かみ）・cute（かわいい）・clear（読みやすい）
- **メインカラー**：ローズ `#e8836a`、ピーチ `#fce8dd`、クリーム `#fdf6f0`
- **フォント**：Nunito（本文）/ Lora（見出し・タイトル）
- CSSは `css/style.css` に一元管理。記事ページも `../css/style.css` で共有

---

## i18n（EN/JA 切り替え）の仕組み

- index.html 内の `<script>` に `i18n` オブジェクトがあり、`en` と `ja` のキーを持つ
- HTML 要素に `data-i18n="キー名"` を付けると `textContent` が切り替わる
- HTML（`<strong>` など）を含む場合は `data-i18n-html="キー名"` を使う
- `placeholder` は `data-i18n-placeholder="キー名"` を使う
- 言語選択は `localStorage` に保存されるのでページをまたいで維持される

---

## 検索の仕組み

- `.post-card` の `data-content` 属性に本文の重要キーワードをスペース区切りで書く
- 検索は **タイトル・抜粋・data-content** の3箇所を対象にする
- タグフィルターと検索は**同時に動く**（AND条件）

```html
data-content="ishiguro books time regret meaningful 読書 時間 本"
```

→ 日本語・英語どちらのキーワードも入れておくと両言語で検索できる

---

## タグフィルタリングの仕組み

- `.post-card` に `data-tags="タグ名"` を付ける（複数の場合はカンマ区切り）
- フィルターボタン・サイドバータグクラウド・heroタグリンクはすべて連動している
- 記事ページのタグは `../index.html?tag=タグ名` にリンクしているので、クリックするとホームページでフィルター済み状態で開く
- `history.replaceState` を try/catch で囲んでいる（file:// プロトコルでのエラー対策）

---

## 注意点・ハマったこと

### file:// で開くとJSエラーになる
`history.replaceState` は `file://` プロトコルで SecurityError になる。
→ try/catch で囲むことで解決。プレビューはサーバー経由（localhost:3000）で確認するのがベター。

### プレビューサーバーの起動
`.claude/launch.json` に設定済み。`python -m http.server 3000` で起動。

### 記事を追加するたびに index.html を手動更新する必要がある
静的サイトのため、index.html の以下を毎回更新する：
1. ブロググリッドのカード
2. i18n 翻訳（en・ja 両方）
3. サイドバーの Recent Posts（最新4件）
4. Stats の記事数
