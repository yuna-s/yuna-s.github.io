## i18n（EN/JA 切替）の仕組み

### index.html
- インライン `<script>` 内の `i18n` オブジェクト（`en` / `ja` キー）
- `data-i18n="key"` → `textContent` を切替
- `data-i18n-html="key"` → `innerHTML` を切替（`<strong>` 等を含む場合）
- `data-i18n-placeholder="key"` → input の placeholder を切替
- 複雑なブロックは `class="lang-en"` / `class="lang-ja"` で show/hide

### 記事ページ（posts/*.html）
- `js/post.js` を読み込む
- EN/JA コンテンツを `<div class="lang-en">` / `<div class="lang-ja">` で分割
- `localStorage` に言語設定を保存（ページ跨ぎで同期）

---

## 検索機能

- 検索対象：タイトル・抜粋・`data-content` 属性
- `data-content` に本文の重要キーワードを日英両方入れる
- タグフィルターと検索は AND 条件で同時動作
- 制約：静的サイトのため全文検索は不可（`data-content` で代替）

---

## タグフィルタリング

- 複数タグ：`data-tags="tech daily life"` のようにスペース区切り
- フィルターボタン・ヒーロータグ・サイドバータグクラウドが連動
- URL パラメータ `?tag=tech` でフィルター状態を共有可能
- タグ内部値は英語で統一（表示テキストのみ i18n で切替）

## AI 共同執筆の開示バナー

記事の `.post-body` 先頭に必ず入れる：

```html
<div class="ai-disclosure">
  <span class="lang-en">🤖 This article was written in collaboration with Claude (AI). The content has been reviewed and edited by the author.</span>
  <span class="lang-ja">🤖 この記事は AI（Claude）と共同で執筆しています。内容は筆者が確認・編集しています。</span>
</div>
```

## 注意点・ハマりポイント

| 問題 | 対処 |
|------|------|
| `file://` で開くと `history.replaceState` が SecurityError | `try-catch` で囲む（対処済み） |
| CSS 変数 `--bg-soft` / `--accent` は `--bg` / `--rose` のエイリアス | `style.css` の `:root` に定義済み |
| 記事追加時に `index.html` の手動更新が必要 | ルール.md のチェックリスト参照 |
| プレビューは `localhost:3000` で確認 | `.claude/launch.json` に設定済み |
